2021/12/19
ttk,ttk_untilを追加
ざっくりとしたとっておきの値を出せます

使い方
 コマンドは半角で、またスペース区切りで設定すること
[]内が引数(かっこ自体は書かない)
()内のものは省略できる(バフなど)

#[comment]
 #から始まるものは行末まで無視されます(コメントに使用)
命令の最後に書いてもよい
ショートカットは「Ctrl + /」
例:
move 125 enemy
#ここからデバフを入れる
例:
move 125 momo #とっておき

set [name] [SPD] ([buff(%)])
開始(start)前にフィールドに出すキャラを設定
SPDの優先度の高いキャラから先に書く
nameにスペースを含めるのは禁止

start
メインモード切り替え
setを書き終わったら記入

end
メインモードモード終了
その後ろにstart、start_sortを書くことでまた別のモードに切り替えられる

move [硬直(%)] [name]
行動
先頭のキャラ以外をnameに記入した場合errorを吐く
moveの代わりにmでも可

action [name] [硬直(%)]
行動
moveと引数の順番が異なる以外同じ
actionの代わりにacでも可

start_sort
ソートモード
end_sortを記入すると誰かのmove_listで書かれた硬直リストがなくなるまで行動する
(他のキャラにまだ硬直リストが残っていても終了)

end_sort
ソートモード終了
これを書かないとソートされない
その後ろにstart、start_sortを書くことでまた別のモードに切り替えられる

move_list [name] [[硬直(%)],[交代後のname,SPD,(buff)],[硬直(%)],{command,argments1,argments2...}...]
行動リスト
move_list nagi [40,{buffset,nagi,24},100,[ren,141,100],100]
のように書く
スペースを空けるのは厳禁
内側にある[]は途中で交代することを示す
{}でコマンドを使用可能
<>で硬直値を直接入力可能
move_listの代わりにmv_lsでも可


----ここからほとんどデバッグしていないので正しく動かない可能性大----

buffset [name] [buff(%)]
buffadd [name] [buff(%)]
buffminus [name] [buff(%)]
バフ設定
それぞれ b,b+,b-と省略可

switch [from_name] [to_name] [SPD] ([buff])
from_nameに交代前のキャラ名、to_nameに交代後のキャラ名を記入
swと省略可

order [name] [name] [SPD]
硬直地直接入力

skillcard [name] [SPD] [硬直] [回数]
スキルカード

ttk [name] [name] [name] ...
ttk_until [n行目]
とっておきカウント
ttkは引数にプレイヤーのキャラ、スキルカードの名前を書きます
ttk_untilは引数に何行目までカウントしたいか書きます
